## 1. MVP 정의

웹 브라우저에서 **웹캠 모션(좌우 기울이기·점프)으로 캐릭터를 조작**해, **혼자서 공을 밀고 굴리며 움직이는** 최소 제품. 상대 플레이어·대전·승패는 없다.

이번 반복의 목표는 **가장 위험한 두 요소 — 실시간 모션 인식과 2D 물리(캐릭터↔공 상호작용) — 가 실제로 동작함을 검증**하는 것이다. 대전·득점·타이머, 온라인 멀티플레이어, 아바타/필터, 모바일·멀티브라우저 최적화는 의도적으로 제외한다.

### 성공 기준 (이 셋이 모두 충족되면 MVP 완료)
1. 사용자가 웹캠 앞에서 몸을 좌우로 기울이고 위로 점프하면, 캐릭터가 좌우 이동·점프한다. 입력→화면 반영 지연이 **100ms 이하**.
2. 캐릭터가 공에 닿으면 공이 자연스럽게 밀려나고 굴러가며, 벽·바닥과 충돌해 화면 안에 머문다.
3. 처음 접속한 사용자가 별도 설명 없이 홈 화면 → 튜토리얼 → 플레이까지 진입해 **공을 움직여 볼 수 있고**, Chrome 데스크톱에서 **30FPS 이상**으로 구동된다.

## 2. 기술 스택

| 영역 | 선택 | 이유 |
|---|---|---|
| 언어/빌드 | TypeScript + Vite | 빠른 개발 서버, 타입 안정성, 번들 분할 용이 |
| 게임 엔진 | Phaser 3 (Arcade 물리) | 2D 스프라이트·입력·씬·물리(중력/충돌/반발) 내장, MVP 개발 속도 우선 |
| 모션 인식 | MediaPipe Tasks `PoseLandmarker` (GPU delegate) | 저지연·고정확 포즈 추정, 브라우저 WASM/GPU 가속으로 100ms 목표 달성 가능 |
| 웹캠 | 표준 `navigator.mediaDevices.getUserMedia` | 플러그인 무설치, 웹 표준 |
| 대상 환경 | Chrome 데스크톱 (최신) | MVP 검증 범위를 한 브라우저로 한정 |

플러그인·네이티브 설치 없이 순수 웹 표준 API와 JS 라이브러리만 사용한다.

## 3. 시스템 아키텍처

### 3.1 디렉터리 구조
```
src/
  main.ts                  # 진입점: Phaser 게임 인스턴스 부트
  motion/
    CameraSource.ts        # 웹캠 스트림 획득, 권한/오류 처리
    PoseDetector.ts        # MediaPipe PoseLandmarker 래퍼 (프레임 → 랜드마크)
    MotionMapper.ts        # 랜드마크 → InputState 변환 (좌우/점프)
    MotionConfig.ts        # 임계값·감도·스무딩 파라미터
  input/
    InputState.ts          # 공용 입력 타입
  game/
    scenes/
      BootScene.ts         # 에셋 로드 + 로딩 진행 표시
      HomeScene.ts         # 시작 화면 (플레이 / 튜토리얼)
      TutorialScene.ts     # 모션 연습 + 건너뛰기
      PlayScene.ts         # 싱글 플레이 루프 (물리·입력·렌더)
    entities/
      Player.ts            # 캐릭터: 좌우 이동·점프·공 충돌
      Ball.ts              # 공: 물리 속성
      Field.ts             # 필드·벽·바닥
    ui/
      HUD.ts               # 안내 텍스트 / 디버그 정보
  config.ts                # 화면 크기, 물리 상수 등 전역 설정
```

### 3.2 데이터 흐름
```
[웹캠] → CameraSource → PoseDetector(독립 추론 루프)
        → MotionMapper → InputState
                       ↓
       PlayScene.update() (매 프레임 InputState 읽음)
                       ↓
        Player 제어 → Arcade 물리(캐릭터↔공↔벽) → 렌더
```

**핵심 설계 원칙: 입력 추상화.** 모션 매퍼는 아래 타입만 채우고, 게임 로직은 입력 출처를 모른다. 다음 반복에서 키보드/AI/네트워크 상대를 추가할 때 같은 타입을 채우는 소스만 더하면 되므로 게임 코드는 변하지 않는다.

```ts
// input/InputState.ts
export interface InputState {
  moveLeft: boolean;
  moveRight: boolean;
  jump: boolean;   // 상승 에지에서만 1회 트리거
}
```

**모션 추론과 게임 루프 분리.** 포즈 추론은 자체 `requestAnimationFrame` 루프에서 돌며 공유 `InputState`만 갱신한다. 추론이 느려져도 게임의 렌더 루프가 멈추지 않는다.

## 4. 기능 상세 명세

### F1. 웹캠 모션 인식

**F1.1 웹캠 접근**
- `getUserMedia({ video: true })`로 스트림 획득.
- 권한 거부·카메라 없음·점유 중 오류를 분기 처리하고, 사용자에게 안내 메시지와 재시도 버튼을 보여준다.
- 완료 기준: 권한 허용 시 비디오 프리뷰가 보이고, 거부 시 명확한 오류 안내가 뜬다.

**F1.2 포즈 감지**
- MediaPipe `PoseLandmarker`를 GPU delegate, `runningMode: "VIDEO"`로 초기화.
- 매 프레임 비디오에서 랜드마크(코·양 어깨·양 골반 등)를 추출.
- 완료 기준: 사람이 화면에 들어오면 어깨·코 좌표가 안정적으로 갱신된다.

**F1.3 좌우 이동 매핑**
- 양 어깨 중심 x좌표(또는 코 x좌표)를 화면 중앙 기준으로 비교.
- 중앙에서 임계값(`MotionConfig.lateralThreshold`)을 넘어 왼쪽이면 `moveLeft`, 오른쪽이면 `moveRight`.
- 좌표에 이동평균 스무딩을 적용해 캐릭터 떨림 방지.
- 완료 기준: 몸을 왼쪽으로 기울이면 좌측, 오른쪽이면 우측 명령이 안정적으로 발생.

**F1.4 점프 매핑**
- 어깨(또는 코) y좌표가 짧은 시간 내 임계값(`jumpThreshold`) 이상 급상승하면 `jump` 1회 트리거.
- 디바운스로 한 번 점프 후 일정 시간 재트리거 금지(연속 점프 폭주 방지).
- `MotionConfig`에서 점프 감도 조정 가능.
- 완료 기준: 제자리에서 위로 뛰면 점프가 1회 발생하고, 가만히 있으면 오발동하지 않는다.

**F1.5 지연·캘리브레이션**
- 추론 1프레임 소요 시간(ms)을 측정해 디버그 오버레이에 표시.
- 시작 시 중립 자세를 기준점으로 캘리브레이션(현재 어깨 중심을 "중앙"으로 설정).
- 완료 기준: 모션 입력→캐릭터 반응 지연이 100ms 이하.

### F2. 싱글 플레이 게임플레이

**F2.1 필드·환경**
- 가로형 필드, 상하/좌우 벽, 바닥 중력.
- 완료 기준: 공과 캐릭터가 벽·바닥과 충돌하고 화면 밖으로 나가지 않는다.

**F2.2 공 물리**
- 원형 바디. 질량·반발(bounce)·마찰·최대 속도 상수를 `config.ts`에 정의.
- 캐릭터·벽과 충돌 시 자연스럽게 튀고 굴러간다.
- 완료 기준: 캐릭터가 공에 닿으면 공이 밀려나가고, 벽에 맞으면 반사된다.

**F2.3 캐릭터 제어**
- `Player`는 `InputState`를 받아 좌우 속도 설정·점프(바닥 접지 시에만) 수행.
- 공과 충돌 시 물리적으로 상호작용(공을 밀어냄).
- 완료 기준: 모션 입력대로 캐릭터가 좌우 이동·점프하고, 공과 부딪치면 공이 움직인다.

**F2.4 리셋 (선택)**
- 공이 멈추거나 구석에 끼었을 때 공을 초기 위치로 되돌리는 리셋 키/버튼 제공.
- 완료 기준: 리셋 시 공이 화면 중앙으로 복귀한다.

> 득점·골대·경기 타이머·승패는 이번 MVP 범위에 포함하지 않는다. (다음 반복)

### F3. UI 흐름 (최소)

**F3.1 화면 전환**
- `Boot → Home → (Tutorial) → Play` 흐름. Play에서 Home으로 돌아가기 가능.
- 완료 기준: 모든 전환이 버튼/종료 이벤트로 매끄럽게 동작.

**F3.2 홈 화면**
- "플레이 시작", "튜토리얼 보기" 버튼을 명확히 배치.
- 완료 기준: 클릭으로 해당 화면 진입.

**F3.3 튜토리얼**
- 좌우 이동·점프 모션을 단계별로 안내하고 실제로 연습(라이브 감지 피드백).
- "건너뛰기" 버튼 상시 노출.
- 완료 기준: 신규 사용자가 조작법을 익히거나 즉시 건너뛸 수 있다.

**F3.4 플레이 화면 정보(HUD)**
- 현재 감지 상태(좌/우/점프)와 추론 지연 등 간단한 안내·디버그 정보를 표시.
- 완료 기준: 사용자가 자신의 모션이 인식되는지 화면에서 확인할 수 있다.

**F3.5 로딩 표시**
- 에셋·모델 로드 중 진행 상태(스피너 또는 진행률) 표시.
- 완료 기준: 로딩 중 빈 화면 없이 진행 피드백이 보인다.

### F4. 성능 (MVP 기준)
- Chrome 데스크톱에서 플레이 중 30FPS 이상 유지.
- 모션 추론과 게임 렌더 루프 분리로 추론 지연이 프레임 드랍으로 전파되지 않게 함.
- 완료 기준: 일반 사양 PC에서 플레이 내내 30FPS 이상.

## 5. 구현 단계 (단계마다 검증 체크포인트)

**0단계 — 셋업**
- Vite + TypeScript + Phaser 3 스캐폴드.
- ✅ `npm run dev`로 빈 게임 캔버스가 Chrome에 렌더.

**1단계 — 게임 코어 (키보드 임시 조작)**
- Field/Ball/Player + Arcade 물리(중력·충돌·반발) 구현.
- 검증 단순화를 위해 캐릭터를 임시 키보드로 조작.
- ✅ 키보드로 캐릭터를 움직여 공을 밀면 공이 굴러가고 벽에 튕긴다.

**2단계 — 모션 인식 (게임과 분리해 단독 검증)**
- CameraSource + PoseDetector + MotionMapper, 디버그 오버레이(감지 상태·지연 ms).
- ✅ 좌우 기울이면 LEFT/RIGHT, 점프하면 JUMP 표시. 지연 100ms 이하 확인.

**3단계 — 통합**
- 캐릭터 입력 소스를 키보드 → 모션으로 교체. 점프 감도·캘리브레이션.
- ✅ 웹캠 모션만으로 캐릭터를 움직여 공을 자유롭게 굴릴 수 있다.

**4단계 — UI 흐름 & 마감**
- Home → Tutorial → Play 연결, 튜토리얼 건너뛰기, 로딩 표시, 30FPS 확인.
- ✅ 신규 사용자가 안내만 보고 공을 움직여 볼 수 있다.
