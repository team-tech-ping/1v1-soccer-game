## 1. 범위

### 1.1 v1 포함 (기능명세서 매핑)
| # | v1 기능 | 명세서 근거 |
|---|---|---|
| ① | **경기 완성**: 90초(상수) 타이머, 승/패/무 판정, 결과 화면 | 영역 2.3 |
| ② | **친구초대 온라인 1v1**: 방 코드/링크 + 서버 권위 실시간 동기화 | 영역 3.2 / 3.3 / 3.4 |
| ③ | **UI 흐름**: Home → 튜토리얼 → 로딩 → 인게임 HUD(점수·시간·이름) | 영역 5.1 / 5.2 / 5.3 / 5.4 |
| ④ | **프라이버시 기본**: 웹캠 on/off 토글 + 기본 아바타 + 닉네임 | 영역 4.1 / 4.2 |

### 1.2 v1 명시적 제외 (다음 반복)
- **빠른매칭 큐**(영역 3.1): v1은 친구초대(방 코드/링크)만. 서버 구조는 큐 확장을 고려해 설계.
- **실시간 필터/애니모지 + 웹캠 영상 스트리밍**(영역 4.3): WebRTC 미디어 스택 필요. v1은 아바타/이름만 상대에게 표시.
- **모바일 터치·반응형**(영역 6.4): v1은 데스크톱 Chrome 기준.
- **크로스브라우저 심화**(영역 6.2, Firefox/Edge/Safari): v1은 Chrome 우선.
- 계정·전적·리더보드, 재연결 유예, 골든골 연장.

### 1.3 성공 기준 (v1 완료)
1. A가 방을 만들어 링크를 공유하면 B가 링크로 접속해 **같은 경기**에 진입한다.
2. 두 사람이 각자 모션(또는 키보드)으로 조작하고 **공·상대·점수·남은 시간이 양쪽 화면에서 일치**한다(서버 권위). 지터가 있어도 렌더가 매끄럽다.
3. 90초 종료 시 양쪽이 **동일한 결과(승/패/무)** 화면을 보고, 재대결/홈 이동이 가능하다.
4. 신규 사용자가 Home→튜토리얼을 거쳐(또는 건너뛰고) 조작법을 익혀 플레이하며, 웹캠 on/off·닉네임을 설정할 수 있다.
5. Chrome 데스크톱에서 플레이 중 30fps 이상.

## 2. 아키텍처

### 2.1 핵심 원리 — 서버 권위(server-authoritative)
서버가 **유일한 진실**이다. 물리 시뮬레이션·골 판정·점수·타이머가 서버에서 돌고, 클라이언트는 ① 자기 모션→추상 입력을 송신하고 ② 서버 스냅샷을 부드럽게 렌더한다. **웹캠 영상은 전송하지 않는다**(v1). 두 플레이어는 대칭적 게스트라 공정하며, 빠른매칭 확장이 자연스럽다.

### 2.2 리포 구조 — 모노레포(npm workspaces)
물리 로직을 서버·클라 공유로 추출한다(중복 없는 단일 구현).

```
motion-soccer-game/                (npm workspaces 루트)
  packages/
    shared/          # 클라·서버 공용
      config.ts          # WORLD_WIDTH, 물리 상수, GOAL_*, MATCH_DURATION_MS(=90000) 등
      types.ts           # InputState, Snapshot, 프로토콜 메시지, PlayerMeta 타입
      sim/
        world.ts         # 권위 월드 생성(바닥/경계/골존/2 플레이어/공)
        playerLogic.ts   # 입력 → 속도·점프 규칙 (기존 Player.update 이관)
        ballLogic.ts     # kick 계산 (기존 Ball.kick 이관)
        step.ts          # 1틱 진행 + 골/점수/타이머/승패 판정
    client/          # Phaser 렌더 + 모션 + 넷 클라이언트 + UI
      motion/            # 기존 그대로
      net/NetClient.ts   # WebSocket, 입력 송신, 스냅샷 수신
      game/scenes/       # Boot / Home / Tutorial / Lobby / Play / Result
      game/render/       # 스냅샷 보간 렌더 전용 엔티티
      ui/                # HUD, 아바타, 설정(웹캠 on/off·닉네임)
      profile/           # 로컬 프로필(localStorage: 닉네임·웹캠공개·아바타)
    server/          # Node 권위 서버
      index.ts           # WS 서버 부트
      RoomManager.ts     # 방 생성/참가/코드 관리 (큐 확장 여지)
      GameRoom.ts        # 매치 1개 = 60Hz 시뮬 루프 + 스냅샷 브로드캐스트
```

### 2.3 기술 선택
| 영역 | 선택 | 이유 |
|---|---|---|
| 전송 | `ws` (경량 WebSocket) | 캐주얼 1v1엔 socket.io 오버헤드 불필요. 재연결은 클라에서 최소 구현 |
| 물리(서버) | `arcade-physics` (헤드리스) | Phaser Arcade 물리를 Node에서 재사용 |
| 모노레포 | npm workspaces | shared/client/server 단일 리포 |
| 프로필 | localStorage | 닉네임·웹캠공개·아바타 저장(영역 4.1.2) |
| 호스팅 | 기존 EC2 + nginx `wss://` | 서버 상시 운영 |

## 3. 프로토콜 (WebSocket, JSON)

### 3.1 클라이언트 → 서버
```ts
{ t: "create", meta }                              // 방 생성 (meta: 닉네임·아바타·웹캠공개)
{ t: "join", code, meta }                          // 방 코드로 참가
{ t: "input", seq, moveLeft, moveRight, jump }     // 입력 (~30Hz)
{ t: "rematch" }                                   // 결과 화면에서 재대결
```

### 3.2 서버 → 클라이언트
```ts
{ t: "created", code, playerId, side }             // side: "left"|"right"
{ t: "joined", playerId, side, opponent: meta }    // 참가 성공(+상대 메타)
{ t: "peerMeta", meta }                            // 상대 닉네임·아바타·웹캠공개
{ t: "start", countdown: 3 }                        // 2명 모임 → 카운트다운 후 킥오프
{ t: "snapshot", tick, timeLeft,                    // 상태 스냅샷 (~20-30Hz)
    ball:{x,y,vx,vy},
    players:[{id,x,y,vx,vy,facing}],
    score:{left,right} }
{ t: "goal", side, score }                          // 득점 이벤트(효과용)
{ t: "end", result:"left"|"right"|"draw", score }   // 90초 종료
{ t: "opponentLeft" }                               // 상대 이탈
{ t: "error", reason }                              // 방 없음/가득참/코드오류
```
`seq`(입력 순번)는 향후 로컬 예측/보정을 위해 포함(v1 미사용). `meta = { nickname, avatarId, camPublic }`.

## 4. 시뮬레이션 (서버, 권위)

- **고정 타임스텝 60Hz**. `arcade-physics` 월드에서 매 틱:
  1. 두 플레이어 최신 입력을 각자 바디에 적용 (`shared/sim/playerLogic`)
  2. 물리 스텝 → 충돌·중력·마찰, **플레이어↔플레이어 충돌**(영역 2.4.2) 포함
  3. 플레이어↔공 충돌 시 kick 계산 (`shared/sim/ballLogic`)
  4. 골존 overlap → 점수 +1, 쿨다운, 공·플레이어 위치 리셋
  5. 타이머 감소, 0이면 `end` 판정(점수 비교 → 승/패/무)
- **스냅샷 20–30Hz 다운샘플**로 브로드캐스트(대역폭 절감).
- **바닥 통과(터널링) 방지**: "월드 경계 = 바닥 윗면 clamp"를 서버 월드에도 적용(MVP와 동일).
- **경기 시간**: `MATCH_DURATION_MS` 상수(v1 기본 90,000ms). 명세서의 "예: 2분"은 예시이므로 상수로 조정 가능.
- **동점**: v1은 **무승부(draw)**. 연장은 다음 반복.

## 5. netcode (클라이언트, 캐주얼)

- **스냅샷 보간(interpolation)**: 받은 스냅샷을 버퍼에 쌓고 **약 100ms 뒤 시점**을 렌더해 두 스냅샷 사이를 보간. 20–30Hz 패킷이어도 60fps로 매끈.
- **공·상대·내 캐릭터 모두** 스냅샷 기반 렌더(권위 일치). 내 입력은 즉시 송신.
- **카메라**: 보간된 공 위치를 따라 스크롤(기존 방식).
- **반응성 보강(스트레치, v1 기본 제외)**: 내 캐릭터가 굼떠 보이면 `seq` 기반 로컬 예측+스냅-보정 추가.

## 6. 화면(씬) 흐름 & UI

```
Boot(로드+진행률) → Home → [Tutorial] → Lobby → Play → Result → (재대결/Home)
```
- **Boot**: 에셋·MediaPipe 모델 로드 진행률 표시(영역 5.2.4), 5초 내 목표.
- **Home**(영역 5.1): "방 만들기" / "코드로 참가"(입력) / "혼자 연습" / **설정**(닉네임·웹캠 on/off·아바타 선택). 첫 방문 시 튜토리얼 유도.
- **Tutorial**(영역 5.3): 좌우·점프 모션을 라이브 감지 피드백으로 연습, **건너뛰기** 상시. 설정에서 재실행 가능.
- **Lobby**(영역 3.2): 방 코드 + **공유 링크**(`?room=CODE`), 상대 대기. 링크 진입 시 코드 자동 입력. 2명 → `start`(3초 카운트다운) → Play.
- **Play**(영역 5.4): 서버 스냅샷 렌더 + **HUD(양쪽 점수·남은 시간 M:SS·양쪽 닉네임·게임상태)** + 각 캐릭터 위 아바타/이름 배지. 모션/키보드 입력 송신.
- **Result**(영역 2.3.3): 승/패/무 + 최종 스코어, "재대결"(양쪽 `rematch`) / "홈으로".

## 7. 프라이버시 / 아바타 (영역 4, v1 기본)

- **웹캠 on/off 토글**(4.1.1): 내 로컬 모션 프리뷰 표시 여부. off여도 모션 인식은 계속 동작(프리뷰만 숨김) — 필요 시 키보드 폴백.
- **프로필 저장**(4.1.2): 닉네임·웹캠공개·아바타를 localStorage에 저장, 다음 접속 시 유지.
- **상대에게 표시**(4.2): 상대 화면엔 **항상 아바타(도형/이모지 실루엣) + 닉네임**. v1은 웹캠 영상 스트리밍을 하지 않으므로 공개/비공개 무관하게 아바타 표시. `camPublic` 플래그는 상대에게 전달만(향후 영상 공유 대비).
- **기본 아바타 커스터마이즈**(4.2.3, 낮음): 색상/이모지 몇 종 선택 — 여유 시 포함, 아니면 단일 기본 아바타.

## 8. 이탈 · 에러 · 성능

### 8.1 이탈/연결
- **경기 중 상대 이탈**: 남은 쪽에 `opponentLeft` → 안내 후 Home 복귀(v1 몰수승 없음).
- **재연결 유예**: v1 범위 밖. 끊기면 방 종료.
- **방 정리**: 둘 다 나가면 서버가 방 파기. 하트비트 핑으로 유령 방 방지.
- **WS 폴백**(영역 3.4.4): v1은 미구현(제외). 실패 시 명확한 에러 후 재시도.

### 8.2 에러
- WS 실패/서버 다운 → "서버에 연결할 수 없어요" + Home.
- 방 없음·가득참·잘못된 코드 → Lobby에서 메시지 후 재시도.
- 카메라·모션 에러 → 기존 키보드 폴백 유지(온라인에서도 키보드 플레이 가능).

### 8.3 성능(영역 6, 기본)
- 무설치 웹 표준만 사용(getUserMedia/WebSocket/WebGL). `requestAnimationFrame` 게임 루프.
- 모션 추론과 렌더 루프 분리(기존) → 추론 지연이 프레임에 전파되지 않음.
- 오브젝트 풀링·텍스처 재사용으로 30fps 유지. Boot 로딩 5초 내.

## 9. 테스트 전략

- **shared/sim 단위**: kick 계산, 골/승패 판정, 타이머·리셋 — 브라우저 없이 순수 함수 검증.
- **서버 통합**: 스크립트 입력 2개를 mock WS로 주입 → 양쪽이 **같은 스냅샷·같은 `end`** 수신, 골/점수/타이머 전이 검증.
- **수동**: 로컬 서버 + 탭 2개, 네트워크 스로틀로 지연 걸어 보간 확인. Home→Tutorial→Lobby→Play→Result 전 흐름 클릭 검증.

## 10. 구현 순서 (위험 먼저)

1. **모노레포 재편 + shared 물리 추출** → 싱글("혼자 연습")이 shared 로직으로 그대로 동작(리그레션 게이트).
2. **경기 완성(로컬)**: 90초 타이머·승패·Result 화면 — shared/sim에 타이머/승패 판정 추가, 로컬에서 먼저 검증.
3. **서버 권위 시뮬 + 2탭 동기화**: 넷코드 핵심(최대 리스크). 입력 송신·스냅샷 보간.
4. **방/코드/링크 + Home/Lobby/Result 흐름**.
5. **UI/프로필**: 튜토리얼·로딩·HUD(이름/시간)·웹캠 on/off·아바타.
6. **EC2 배포(wss) + 실 지연에서 보간·30fps 확인**.
